# 前端设计文档（daily_paper2）

## 1. 目标与范围
本设计文档用于指导 `website` 目录下的前端实现，覆盖以下功能：
1. 左侧侧边栏 + 右侧内容区的整体框架。
2. 三个子页面：`论文日报`、`论文探索`、`设置`。
3. 页面交互逻辑、状态流转、接口契约与异常/空状态处理。

说明：本文档只定义设计，不包含任何代码实现。

## 2. 信息架构

### 2.1 全局布局
- 页面结构：左右分栏。
- 左侧：固定侧边栏（导航）。
- 右侧：主内容区（根据当前菜单切换子页面）。

### 2.2 导航菜单
- 菜单项（自上而下）：
1. 论文日报
2. 论文探索
3. 设置
- 当前选中项高亮。
- 支持路由直达（例如刷新后仍停留在当前页）。

### 2.3 推荐路由
1. `/daily-report`
2. `/paper-explore`
3. `/settings`

## 3. 视觉与响应式布局

### 3.1 桌面端（>= 1024px）
- 侧边栏宽度：240px（固定）。
- 主内容区：自适应剩余宽度，内部最大宽度建议 1440px。
- 页面内卡片、表格、Markdown 区块采用统一间距体系（8/12/16/24）。

### 3.2 平板/移动端（< 1024px）
- 侧边栏折叠为顶部抽屉菜单（汉堡按钮触发）。
- 主内容全宽显示。
- 表格在移动端支持横向滚动，不强制压缩列内容。

## 4. 数据映射（与现有表结构对齐）

### 4.1 `papers`（论文元信息）
核心字段：
- `id`
- `title`
- `published_at`
- `source`
- `online_url`
- `pdf_url`
- `abstract`
- `extra`

前端展示映射：
- 标题：`title`
- 关键词：优先 `extra.keywords`（若后端提供），否则显示 `-`
- 发表时间：`published_at`
- 论文源：`source`

### 4.2 `activity`（论文活动）
核心字段：
- `id`（对应 `papers.id`）
- `recommendation_records`
- `user_notes`
- `ai_report_summary`
- `ai_report_path`

前端用途：
- “添加笔记”读写 `user_notes`
- “查看详情”中展示 AI 摘要、报告路径、推荐记录

### 4.3 `report`（日报）
核心字段：
- `id`
- `report_date`
- `generated_at`
- `related_paper_ids`
- `local_md_path`

前端用途：
- 按日期查询日报
- 展示该日报关联论文列表
- 渲染对应 Markdown 内容

## 5. 子页面设计

## 5.1 论文日报页

### 5.1.1 布局结构（自上而下）
1. 页面标题区：`论文日报`
2. 查询区：日期选择器（默认当天）+ 查询按钮
3. 日报信息区：显示 `report_date` / `generated_at` / `id`
4. 关联论文表格区：结构与“论文探索”一致
5. Markdown 渲染区：显示日报正文

### 5.1.2 交互逻辑
1. 初次进入页面：
   - 日期默认当天（本地时区日期）。
   - 如果为空显示生成日报按钮。
2. 用户切换日期后点击查询：
   - 重新拉取该日期日报。
3. 查询成功：
   - 渲染日报基础信息。
   - 根据 `related_paper_ids` 拉取并显示论文表格。
   - 读取并渲染 `local_md_path` 对应 Markdown 文本。

### 5.1.3 空状态与异常状态
1. 当天（或所选日期）无日报：
   - 显示空状态文案：`该日期暂无日报`。
   - 不显示 Markdown 内容区域（显示空占位）。
2. 日报存在但关联论文为空：
   - 不显示表格
3. 日报存在但 Markdown 文件不存在/读取失败：
   - 显示警告提示：`日报内容文件不可用`。
4. 网络失败：
   - 显示错误提示并提供“重试”。

### 5.1.4 Markdown 渲染要求
- 支持标题、列表、代码块、表格、链接、公式。
- 链接默认新窗口打开。
- 对 HTML 注入进行安全过滤（避免 XSS）。

## 5.2 论文探索页

### 5.2.1 布局结构
1. 页面标题区：`论文探索`
2. 可选工具栏：关键字搜索、来源筛选、时间筛选（建议）
3. 数据表格区（核心）

### 5.2.2 表格字段
固定列：
1. 操作列（图标组）
2. 标题
3. 关键词
4. 发表时间
5. 论文源

说明：
- 表格数据来自 `papers` 与 `activity` 的合并视图（按 `papers.id = activity.id` 关联，`activity` 可为空）。
- 支持分页（建议默认每页 20）。

### 5.2.3 操作列设计（图标 + 悬浮提示）
每行显示 4 个操作图标，鼠标悬浮显示文字：
1. 查看详情
2. 阅读原文
3. AI解读
4. 添加笔记

### 5.2.4 操作行为定义
1. 查看详情
   - 点击后打开弹窗。
   - 以卡片形式展示：
     - id、标题、作者、摘要、来源、发表时间、链接
     - 推荐记录、用户笔记、AI 摘要、AI 报告路径
2. 阅读原文
   - 点击后新窗口打开 URL。
   - `online_url`。
   - 若 URL 为空，按钮置灰并提示 `无可用链接`。
3. AI解读
   - 点击后调用后端接口，提交异步深度解析任务。
   - 成功后提示 `任务已提交`，并返回 `task_id`。
   - 当前行展示“进行中”状态（可选小徽标），阻止再次点击。
4. 添加笔记
   - 点击后弹出输入框（多行文本）。
   - 默认回填当前 `user_notes`。
   - 提交成功后更新 `activity.user_notes`，并刷新当前行数据。

### 5.2.5 异常处理
1. AI任务提交失败：
   - 弹出错误提示，不改变行状态。
2. 笔记保存失败：
   - 保留输入内容，提示失败原因并支持重试。
3. 详情加载失败：
   - 抽屉内显示错误态 + 重试按钮。

## 5.3 设置页

### 5.3.1 布局结构
1. 页面标题区：`设置`
2. 异步任务列表区（重点）

### 5.3.2 任务列表内容
建议列：
1. `task_id`
2. 任务类型
3. 关联信息（日报日期/解析论文id）
4. 状态（queued/running/success/failed）
5. 启动时间
6. 已运行时长

默认显示“正在运行”的任务；可切换查看全部标签页。

### 5.3.3 日志弹窗
- 点击任务行后打开弹窗，展示该任务 `stdout` 日志。
- 日志区要求：
1. 等宽字体
2. 自动滚动到最新行（可手动关闭自动滚动）
3. 支持定时刷新或流式更新
4. 显示最后更新时间

### 5.3.4 任务与日志状态
1. 无运行任务：显示空状态。
2. 任务存在但暂无日志：显示 `暂无日志输出`。
3. 日志加载失败：显示错误态 + 重试按钮。

## 6. 前后端接口契约（建议）

说明：后端当前未实现 Web API，本节定义前端所需最小接口集合，便于后续联调。

### 6.1 日报相关
1. `GET /api/reports/by-date?date=YYYY-MM-DD`
   - 返回：单日报对象（含 `related_paper_ids`、`local_md_path`）
2. `GET /api/reports/{report_id}/papers`
   - 返回：该日报关联论文列表（字段同探索表）
3. `GET /api/reports/{report_id}/markdown`
   - 返回：Markdown 原文（字符串）

### 6.2 论文探索相关
1. `GET /api/papers/explore?page=1&page_size=20&keyword=&source=`
   - 返回：`papers + activity` 合并分页结果
2. `GET /api/papers/{paper_id}/detail`
   - 返回：`paper` + `activity` 详情
3. `PATCH /api/activities/{paper_id}/notes`
   - 入参：`{ "user_notes": "..." }`
   - 返回：更新后的 `activity`
4. `POST /api/papers/{paper_id}/ai-interpret`
   - 入参：可选解析参数
   - 返回：`{ "task_id": "...", "status": "queued" }`

### 6.3 异步任务相关（设置页）
1. `GET /api/tasks?status=running`
   - 返回：任务列表
2. `GET /api/tasks/{task_id}`
   - 返回：任务详情
3. `GET /api/tasks/{task_id}/logs?offset=0`
   - 返回：日志分片 + next_offset

## 8. 关键交互时序

### 8.1 “论文日报”查询时序
1. 用户选择日期并触发查询。
2. 前端请求日报数据。
3. 若无数据，进入空状态。
4. 若有数据，继续请求关联论文与 Markdown。
5. 全部返回后渲染页面。

### 8.2 “AI解读”时序
1. 用户点击某行“AI解读”。
2. 前端提交异步任务请求。
3. 后端返回 `task_id`。
4. 前端提示成功并刷新设置页任务列表（或本地插入任务）。
5. 用户可在设置页查看实时日志。

### 8.3 “添加笔记”时序
1. 用户点击“添加笔记”。
2. 弹窗输入文本并提交。
3. 后端更新 `activity.user_notes`。
4. 前端关闭弹窗并刷新行数据。

## 9. 可用性与一致性要求
1. 所有按钮具备禁用态（加载中、无权限、无数据）。
2. 所有图标按钮必须有 tooltip 文案。
3. 所有表格空状态文案统一。
4. 异步请求统一展示加载态和错误态。
5. 打开新窗口的行为统一使用安全模式（`noopener/noreferrer`）。

## 10. 验收清单（用于实现后自测）
1. 左侧导航能稳定切换三页，刷新后路由保持。
2. 论文日报默认加载当天，切换日期后数据正确更新。
3. 无日报日期能正确显示空状态。
4. 日报页论文表格结构与探索页一致。
5. 探索页每行 4 个操作图标均可触发对应行为。
6. 阅读原文能在新窗口打开在线 PDF/页面。
7. AI解读能提交异步任务并在设置页可见。
8. 添加笔记可写入并回显 `activity.user_notes`。
9. 设置页可展示运行中任务，点击可查看 stdout 日志。
10. 主要异常场景（接口失败、数据缺失）均有明确反馈。
